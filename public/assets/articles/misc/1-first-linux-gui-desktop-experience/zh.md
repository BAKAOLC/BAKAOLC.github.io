## 起因

Windows 的操作实在是有点不让我舒服了，尤其是我切换到超宽屏显示器之后，明明屏幕这么宽，自带的那个贴靠功能却永远只想着把窗口塞成左右两半，或者最多四等分，我想搞个单纯的左中右三栏的专注布局都不行。  
然后我试着用 FancyZones，唉，虽然能用但总是有些窗口有奇怪的问题，而且 windows 的边框会透一条缝隙出来，太难看了。  
之后我还试着用了用其他的平铺窗口管理器，比如那些第三方工具或者扩展。恶心啊，恶心，恶心死了！完全不符合我的使用习惯。为啥非得自动瞎 jb 调整窗口位置和大小呢？非要全部都动态调整，我就想要个静态的布局有这么难吗？

最后我实在受不了了，于是我决定使用我之前一直想尝试的 Niri 窗口管理器，它的设计理念太符合我的需求了。当然，这也就代表我必须用 Linux 系统了，毕竟 Windows 上是没法用 Niri 的。  
不过还好，现在 Linux 的硬件支持和软件生态都已经相当不错了，应该不会有太大问题。而且我个人来说本身也比较喜欢自己搞些自定义的系统逻辑脚本，捣鼓 Linux 反而应该更适合我（大概）。

### 为什么选择 Arch Linux

很简单，因为 Arch Linux 的滚动更新和极简安装让我可以从零开始，完全按照我的需求来配置系统。  
而且其他的发行版大多都预装了一堆我不需要的软件和服务，反而增加了系统的复杂性。
而且 AUR 很方便，我感觉比其他发行版的软件源要舒服很多。

## 系统环境

- **发行版**: Arch Linux x86_64
- **内核**: Linux 6.18.7-arch1-1
- **窗口管理器**: niri 25.11 (Wayland) + DankMaterialShell 1.2.3
- **Shell**: zsh 5.9 + powerlevel10k
- **终端**: kitty 0.45.0
- **包管理**: pacman, paru, flatpak
- **交互增强**: fzf-tab + fast-syntax-highlighting + yazi
- **现代命令替换**: eza, bat, fd, rg, dust, delta, btop, tldr, zoxide, fastfetch

我目前配置下来的系统环境大致如上所示，当然随着时间的推移可能会有所变化。

## 体验

### niri 窗口管理器

**爽，很爽，爽爆了，爽死我了，超级无敌爽**  
用下来才几天我已经快忘了 Windows 的操作了。  

目前最大的缺点：现在切回 windows 的时候觉得 windows 的操作更难受了。

### Shell 和终端

kitty 很好用，zsh 配合 powerlevel10k 也很舒服。  
不过快捷键和一些操作我还是不太习惯，用 windows 习惯了。  
经常会按歪，比如我经常习惯性去按 ctrl+left/right 切换单词、ctrl+backspace/delete 删除单词之类的，然后就会直接把快捷键用文本输入出来了，蛋疼。  
我还是不太理解为什么默认要把没有绑定的快捷键直接当成文本输入，这样设计真的好吗？

### 软件生态

基本目前来说我需要用到的软件都有，然后也可以用 proton 来运行一些 Windows 软件。  
除了特定领域用的软件外别的基本没啥问题。
Adobe 啥的在 windows 我都没去用了，所以也没啥影响。
