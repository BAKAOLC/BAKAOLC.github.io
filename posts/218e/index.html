Github简易使用教程

本篇文章内提到的软件均可在该百度网盘中下载：r56w

本篇文章面向零基础的用户编写，因此不会进行过于全面的讲解，仅提供基础的操作，以便快速的上手。

准备工作

在使用 Github 前，我们首先需要注册一个账户
具体怎么注册在这我就不进行过多说明了，前往官网自己注册一个就行

对于需要进行文本编辑的，我在这推荐安装 Notepad++

在之后，我们需要安装一个 Git 用于方便快捷的进行各种操作：下载地址

安装时，在 Select Components 步骤请取消掉 Git GUI Here 的复选框，因为Git提供的这个窗口界面并不好用，在下面会提供更好的选择。
而在 Choosing the default editor used by Git 步骤，如果安装了Notepad++，可以将此项修改为 Use Notepad++ as Git’s default editor
除此之外一路Next安装即可

由于 Git 是使用命令行进行操作的，对于大部分基础用户来说，这是十分困难的操作。因此，我们可以安装 TortoiseGit 来辅助操作，安装了这个工具后，我们便可以通过右键菜单完成 Git 的各项操作，十分方便：下载地址

它还提供了中文语言包，对于英文比较差的用户来说，这个实在是太方便了

安装完 TortoiseGit 之后，在桌面或任意文件夹 右键 -> TortoiseGit -> 设置 进入设置界面
之后点击进入 Git 设置界面，选择 全局，取消掉 用户信息 的 名称 与 Email 的继承复选框，并填写你注册的 Github 账户信息，之后点击 确定 保存配置

完成以上操作后，基础的准备工作就都完成了。

将 Github 仓库克隆至本地

该部分内容默认用户已完成准备工作中的配置操作

通过命令行进行克隆

准备工作完成后，我们要对 Github 上的仓库进行操作，我们首先需要对目标仓库执行 Fork 操作（进入目标仓库地址，点击右上角的 Fork 按钮，会自动继承目标仓库到自身账户中创建一个对应的仓库）
之后，在自身账户Fork出的仓库中，点击 Clone or Download 按钮，并复制内部的链接，之后即可在本地进行克隆操作了

通过 TortoiseGit 进行克隆

在本地电脑中你想要保存的位置 右键 -> Git克隆…，在 URL 处填写你刚刚复制出的链接地址，点击 确定 即可自动开始克隆文件到本地。
克隆完成后，进入到克隆出的本地仓库文件夹中 右键 -> TortoiseGit -> 设置 进入设置界面
之后点击进入 Git - 远端 设置界面，在 远端 处输入 upstream 并在 URL 处填写目标仓库的克隆链接，再点击 添加/保存(A) 按钮来创建远端目标，用于对目标仓库保持更新，再点击 确定 保存配置
